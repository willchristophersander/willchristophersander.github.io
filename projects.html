Robot Vacuum & Plumbing Automation

Personal Project · Home Automation

Status: In Progress Source: Closed Source
Python C IoT Hardware Robotics

In Progress automation for my family’s X30 Ultra focused on both plumbing and central-vacuum dust handling, using a Raspberry Pi, custom programming, 3D printed TPU parts, and online-sourced pumps. In parallel, I am developing central-vacuum integration for the base dustbin using 3D printed interfaces, automated gates, and a custom Dreame-compatible dustbin design. The goal is to replace failure-prone cheap pump kits with a system that better handles gray water and automatically performs periodic clean-water flushing to maintain pump health. The current system works, and I am actively rewriting parts of the working Python control code into C for better efficiency while continuing safety and reliability iteration. The system also automatically refills clean water for both the robot and base. Because the X30 Ultra base automatically doses detergent, the mopping workflow is largely hands-off aside from refilling detergent every few months and occasional baseboard cleaning.

  • Integrates pressure-sensing hardware from China to detect dock dump requests
  • Automatically turns on pumping when the robot base requests a water dump
  • Current system works while ongoing iterations focus on reliability and safety
  • Includes teardown-driven research on Dreame X30 base behavior and validation testing
  • Covers TPU print prototyping, plumbing research, Chinese translation/navigation for sourcing, and custom part selection
  • Actively developing central-vacuum dust handling via 3D printed parts, automated gates, and a custom Dreame-compatible dustbin

Evolutionary Robotics Cumulative Project

UVM · CS3060 Evolutionary Robotics

Status: In Progress Source: Closed Source
Python Machine Learning Neural Network Robotics

Cumulative course project developed in CS3060 under Joshua Bongard, a highly respected researcher in evolutionary robotics. The project used PyBullet to build a simulation-based evolutionary robotics workflow and combined weekly deliverables into a full experiment system involving neural-network and robotics components. As summarized by the course: Over the final four weeks of the semester, each student used their software system to perform an evolutionary robotics experiment, submitted weekly deliverables, handed in a written report, and gave an oral presentation during the exam period.

  • Implemented an iterative software pipeline for evolutionary robotics experiments
  • Used simulation outputs to compare robot designs and controller performance
  • Delivered a final integrated project with experiment results and presentation artifacts
  • Development progress video shows PyBullet-based experimentation
